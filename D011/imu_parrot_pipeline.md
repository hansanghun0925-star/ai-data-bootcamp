# AI 데이터 인텔리전스 부트캠프 과제: 레딧 r/OnePiece '이무 = 앵무새론(Imu = Parrot)' 화제성 분석 파이프라인

## 1. 개요

이 프로젝트는 레딧의 원피스 커뮤니티 **r/OnePiece** 게시글을 수집·분석하여, 2026년 7월 15일경 한국 팬 커뮤니티에서 제기된 **"이무(Imu)의 정체는 앵무새(parrot)"** 가설이 레딧에서 언제, 얼마나 확산됐는지를 정량적으로 추적하는 것을 목표로 합니다. 특정 시점 전후의 언급량 변화(시계열)와 커뮤니티 전체에서의 점유율을 분석하며, 데이터 수집·정제·분석·시각화 전 과정을 하나의 재현 가능한 파이프라인으로 구현했습니다. 분석은 Jupyter Notebook 및 단일 Python 스크립트 두 형태 모두로 실행할 수 있습니다.

---

## 2. 데이터 수집 과정 (Data Collection)

- **수집 대상 및 형태**: r/OnePiece의 게시글을 insane-search를 이용해 얻은 서로 다른 두 기간 창으로 내보낸 CSV 2종을 사용합니다. `reddit_onepiece_9days.csv`(999행)와 `reddit_onepiece_16days.csv`(2,431행)이며, 9일치는 16일치에 완전히 포함되는 부분집합입니다.


- **수집 예절(rate limiting)**: 자동 수집 시 서버 부담을 최소화하기 위해 요청 사이에 `time.sleep()`으로 대기 간격을 두고, 대상 사이트의 `robots.txt`를 사전에 확인하여 `Disallow` 경로(예: `/api/`, `/admin/`)를 회피합니다. `User-Agent`는 정직하게 명시합니다.

- **수집 범위의 한계**: 레딧 검색 API는 한 번에 최대 약 1,000건까지만 반환하므로, "모든 게시글"이 아닌 **정렬 기준 상위 N건**을 수집하는 방식이 현실적입니다. 본 분석의 원자료도 이 제약 위에서 확보된 스냅샷입니다.

- **수집된 컬럼**: `작성시간(UTC)`, `제목`, `공감수`, `댓글수`, `작성자`, `링크` (6개 컬럼)

---

## 3. 데이터 처리 과정 (Data Processing)

- **병합 및 중복 제거**: 두 CSV를 `pd.concat()`으로 행 방향 병합한 뒤, 게시글마다 고유한 **`링크`를 키로 `drop_duplicates()`** 를 수행합니다. 링크는 게시글 고유 ID를 포함하므로 제목보다 안전한 중복 판별 기준입니다. (병합 3,430행 → 중복 제거 2,431행)

- **원본 컬럼 처리 방식**:

| 원본 컬럼 | 의미 | 처리 |
|---|---|---|
| `작성시간(UTC)` | 게시 시각(문자열) | `pd.to_datetime()` → `tz_localize('UTC')` → `tz_convert('Asia/Seoul')`로 datetime·KST 변환 |
| `제목` | 게시글 제목 | `.str.lower()`로 소문자 통일 후 키워드 분류의 입력으로 사용 |
| `공감수` | 추천 수(정수) | 정수형 유지, 앵무새론 게시글 반응 분포 분석에 활용 |
| `댓글수` | 댓글 수(정수) | 정수형 유지 |
| `작성자` | 작성자 ID | 참고용 유지 |
| `링크` | 게시글 URL | 중복 제거의 키로 사용 |

- **시간대 정규화**: 가설이 **한국 커뮤니티**에서 7월 15일에 제기됐으므로, 날짜 경계를 정확히 하기 위해 UTC를 KST(UTC+9)로 변환한 뒤 `.dt.normalize()`로 '그 날의 자정' 기준 일자 컬럼을 생성합니다.

- **타입 확인**: 전처리 후 `df.dtypes`와 `df.isna().sum()`으로 각 컬럼의 타입과 결측치를 점검합니다. 특히 `작성시간`이 datetime64인지, `공감수/댓글수`가 정수인지 확인합니다.

- **앵무새론 분류(라벨링)**: 제목 기준으로 다음 규칙을 적용합니다.
  - **앵무새론 = `parrot` 포함 OR (`imu` AND `bird` 동시 포함)**
  - `parrot`은 원피스 맥락에서 거의 쓰이지 않는 단어라 등장 시 앵무새론일 확률이 매우 높으며(전수 검증 결과 오탐 0), `imu`+`bird` 조합은 'parrot'을 쓰지 않고 우회 표현한 글(예: "IMU bird theory")까지 포착합니다.
  - `imu` 단독은 앵무새론이 아닌 **일반 이무 논의**이므로, 이는 별도 지표(이무 언급량)로만 사용합니다.

- **분류 방법의 한계**: 원자료에 **본문(selftext)이 없고 제목만** 존재하므로, 제목에 키워드가 없는 앵무새론 글은 누락됩니다. 따라서 산출된 수치는 **실제 규모의 하한선**이며, 절대량보다 시간에 따른 **추세**를 신뢰합니다.

## 4. 데이터 분석 및 활용 (Output & Visualization)

- **분석 질문**: 앵무새론은 7월 15일을 전후로 언급량이 어떻게 변했는가? 원피스 전체 게시글 대비 점유율은 얼마인가? 다른 커뮤니티 주제(코스프레·팬아트·이론 등)와 비교하면 어느 정도 체급인가?

- **확정 기준(불연속성 검정)**: 7/15 이전 앵무새론 언급이 사실상 0이고 이후 급증하면, 이는 배경 잡음이 아니라 **특정 시점에 발생한 사건**으로 확정합니다. (분석 결과: 이전 0건 → 이후 21건, 기준 충족)

- **시각화 4종** (`imu_parrot_eda.py`, 2×2 대시보드):
  1. 일별 게시글 수 — 앵무새론 vs 이무 전체(선그래프, 7/15 기준선 표시)
  2. 앵무새론 점유율 추이 — 전체 원피스 게시글 대비(면적그래프)
  3. 7/15 전후 누적 건수 비교(막대그래프)
  4. 앵무새론 게시글 공감수 분포(히스토그램)

- **주제별 점유율 비교** (`topic_share_bar.py`): 게시글을 콘텐츠 종류(앵무새론·코스프레·팬아트·이론/떡밥·만화/스포·애니·기타)로 **배타적 배정**하여 점유율을 가로 막대그래프로 비교합니다. 한 글이 한 주제에만 속하도록 우선순위 배정하여 점유율 합이 100%가 되도록 설계했습니다.

- **데이터 명료성**: 정제 전후 데이터를 저장해 재현·검증이 가능하며, 결과 그림은 `imu_parrot_analysis.png`, `topic_share_bar.png`로 저장됩니다.

## 5. 사용 방법

1. **Python 설치**: Python 3.10 이상이 설치되어 있어야 합니다.

2. **패키지 설치**: 터미널에서 아래 명령어(`myenv` 환경)를 입력하여 필요한 라이브러리를 설치합니다.

   ```bash
   pip install pandas numpy matplotlib seaborn
   ```

3. **데이터 배치**: `reddit_onepiece_9days.csv`, `reddit_onepiece_16days.csv`를 스크립트와 **같은 폴더**에 둡니다.

4. **실행**: 다음 두 스크립트를 실행합니다.

   ```bash
   python imu_parrot_eda.py      # 시계열·점유율·전후비교 대시보드
   python topic_share_bar.py     # 주제별 점유율 막대그래프
   ```

   (Jupyter Notebook에서는 셀을 위에서부터 순서대로 Run All 하면 됩니다.)

5. **결과 확인**: 콘솔에 집계 표와 결론 요약이 출력되고, 그림은 `imu_parrot_analysis.png` / `topic_share_bar.png`로 생성됩니다.

## 주요 특징

- **시간대(KST) 정규화**: 한국 커뮤니티발 가설임을 반영해 UTC를 KST로 변환, '7월 15일' 경계를 정확히 처리했습니다.

- **검증된 분류 규칙**: 앵무새론 라벨을 전수 육안 검증(오탐 0)하여 신뢰도를 확보했으며, `imu` 단독(일반 논의)과 앵무새론(특정 이론)을 명확히 분리했습니다.

- **정직한 한계 명시**: 제목만으로 분류하는 데이터 특성상 수치를 '하한선'으로 규정하고, 절대량이 아닌 추세를 신뢰 대상으로 삼았습니다.

- **적절한 시각화 선택**: 크기 비교엔 막대, 시간 변화엔 선/면적그래프를 사용했으며, '점유율' 표현의 정당성을 위해 주제를 배타적으로 배정했습니다.

- **한글 폰트 이슈 해결**: seaborn이 `font.family`를 초기화하는 문제를 고려해, `sns.set_style()`을 먼저 적용한 뒤 한글 폰트(`NanumGothic`)를 지정하도록 순서를 조정했습니다.

- **재현 가능성**: 노트북/스크립트를 실행하면 수집 이후 정제·분석·시각화 전 과정이 재현되며, 정제 전후 데이터가 보존됩니다.
